Grid = "layout first"

  • grid-template-columns: 1fr 1fr 1fr

    This is a grid container.
    Each cell is set to occupy 1fr which means that even if the content of one cell is greater than the others, the width of the cell remain unaffected.

  • grid-template-columns: repeat(3,minmax(min-content,auto))

    This is also a grid container.
    Each cell here behave based on it's content, giving them a more dynamic behavior.

  • @media(max-width: 767px) grid-auto-rows: auto

    This is also a grid container. The difference with this container is a media query that change from columns to rows at smaller screens.
    Each cell here behave based on it's content, giving them a more dynamic behavior.

Flexbox = "content first"

  • flex-grow: 0; flex-shrink: 1; flex-basis: auto

    This is a flexbox with the default setting.
    These boxes will shrink as needed to fit the content into the parent container.

  • flex-grow: 1; flex-shrink: 0; flex-basis: 0

    These boxes mimic the behavior of our first grid.
    Flexing this much must grow muscles, right?

  • @media (max-width: 767px) flex-direction: column

    These boxes are part of the responsive flexbox. The flex-direction is changed when the screen size is reduced.
